## 자바스크립트란

자바스크립트는 **웹 페이지를 동적으로 만들어주는 프로그래밍 언어**이다.

<br />
건물로 비유하자면

HTML은 웹 페이지의 기본 뼈대이다. 구조를 정의한다. div, p 등의 태그로 웹 페이지의 설계도를 구성한다.

CSS는 웹 페이지의 인테리어이다. 스타일을 지정한다. color, margin, padding 등의 속성으로 웹 페이지를 꾸민다.

JavaScript는 웹 페이지의 여러 시스템이다. 엘레베이터, 난방 시스템과 같이 동작을 구현한다. 버튼 클릭 시 동작, 동적 데이터 통신 등 사용자와의 상호작용을 처리한다.

## 자바스크립트의 표준화

넷스케이프의 넷스케이프 내비게이터와 마이크로소프트의 인터넷 익스플로러는 웹 브라우저 경쟁상대였다.

넷스케이프가 자바스크립트를 내놓자 마이크로소프트는 자바스크립트 + 자사 문법을 적용하여 JScript를 내놓았다.

문제는 두 회사가 자사 브라우저의 시장 점유율을 높이기 위해서 자사 브라우저에서만 동작하는 기능들을 마구 추가하기 시작한 점이다.

<br />
이로 인해 **크로스 브라우징 이슈**가 발생하기 시작했다. 브라우저A에서 잘 작동하던
자바스크립트가 브라우저B에서 정상적으로 동작하지 않는다. 모든 브라우저에 호환되는
웹페이지를 개발하기가 어려워졌다.
<br />
크로스 브라우징 이슈로 인해 자바스크립트 표준화의 필요성이 제기되었다.

그렇게 탄생한 표준화된 자바스크립트가 `ECMAScript`이다.
줄여서 'ES + 버전'으로 불린다.

HTML5와 함께 출현한 ES5가 제정되고 나서야 표준화 문제가 다소 해결되었다.
let/const, 화살표 함수, 템플릿 리터럴, 비구조화 할당, 스프레드 문법 등이 들어간 ES6가 아마 익숙할 것이다.

<br />
그렇다고 ECMAScript = JavaScript는 아니다.

ECMAScript는 언어 자체의 값, 타입, 객체, 함수 등 핵심 문법을 규정한다.

자바스크립트는 ECMAScript를 근간으로 브라우저가 별도로 지원하는 Web API (DOM, Canvas, fetch, Web Storage 등)까지 포함한다.

ECMAScript ⊂ 자바스크립트인 셈이다.

## 자바스크립트의 특징

자바스크립트는 3가지 특징이 있다.

### 1. 웹 브라우저에서 동작하는 프로그래밍 언어다.

자바스크립트는 브라우저 내에서 실행된다.

자바스크립트로 UI와 상호작용하고, DOM을 조작하여 웹 페이지에 보이는 내용을 동적으로 변경할 수 있다.

### 2. 인터프리터 언어다.

> 참고) 컴파일러 언어 vs 인터프리터 언어

자바스크립트는 코드를 미리 컴파일하지 않고 실행 시점에서 한줄씩 해석해서 실행한다.

인터프리터 언어는 코드 실행 속도가 비교적 느리다는 단점이 있다.

하지만 대부분의 모던 브라우저 내의 자바스크립트 엔진은 인터프리터와 컴파일러의 장점을 결합해서 단점을 극복해서 실행 속도가 빠르다.

명시적으로 컴파일 과정을 거치지는 않지만 일부 소스코드를 컴파일해서 실행한다.

### 3. 멀티 패러다임 프로그래밍 언어다.

자바스크립트는 함수형, 명령형, 객체지향 프로그래밍을 지원하는 멀티 패러다임 언어이다.

프로그래밍 패러다임은

다시 말해 자바스크립트는 문제 해결의 여러 방법론을 적용할 수 있는 언어이다.

<br />
이는 자바스크립트의 역사와 관련있다.

자바스크립트의 초기 설계 철학은 간단한 스크립트 언어였다.
복잡한 언어를 만드는 대신 러닝커브가 낮은 간단한 언어를 만드는게 브랜든 아이크의 목표였다.

언어적 유사성을 유지하려고 했다.

Java의 객체지향 패러다임은 계승하고자 했다.
